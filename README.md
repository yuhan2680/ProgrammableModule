# 基础玩法

玩家操作一个模块化的机器人完成每个关卡，在每个关卡通关后可获得新的模块和指令

游戏分为两种模式：“编程”和“手动”。（之后或许可以加个“混合”模式，不过先把编程模式做出来之后再考虑另外两个）

（之后可以考虑加上多人对战模式，不过等其他都做完了再说）

## 机器人

在开局仅拥有移动模块和移动指令，后面会解锁越来越多的模块和指令，玩家可以自行组装模块并对机器人进行编程。每一关会限定能够携带的模块数量，相同模块可叠加以获得更强的效果。

模块同时起到库的作用，模块将会为机器人提供更多样的函数，例如有攻击模块，则会有一个 `attack` 函数可以调用。

有一次性模块，在使用完毕一次后就会自动消失，其后相关函数也无法执行

## 编程

默认的模式，玩家可编程让机器人按照指令进行移动。

编程语言为解释型语言，每次执行都会更新机器人的状态，下一次执行时将会以此状态执行

使用强类型的类kotlin的格式（强制换行不能使用分号），但是解释器需要自己实现，以控制语言功能随着游戏进程的解锁

机器人的逻辑由两个主函数实现，即 `init()` 和 `tick()` 函数。

`init` 函数在游戏开始时调用，用于初始化机器人的状态。

`tick` 函数在游戏进行中不断调用，用于更新机器人的状态。

## 手动

手动控制机器人，可自行设置键位。
